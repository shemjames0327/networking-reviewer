1.2 — Network Components
HOST ROLES

Servers provide info to end devices (email, web, file). Clients send requests to retrieve info. Every computer on a network is a host or end device.

PEER-TO-PEER (P2P)

Device can be both client and server. Only for very small networks.

Advantages
Easy setup · Less complex · Low cost
Disadvantages
No central admin · Insecure · Not scalable · Slow

INTERMEDIARY DEVICES

Interconnect end devices. Examples: switches, wireless APs, routers, firewalls. Also: regenerate signals, maintain pathway info, notify of errors.

END DEVICES

Where a message originates from or is received. Data flows through the network from end device to end device.

NETWORK MEDIA

Copper — electrical impulses. Fiber-optic — pulses of light. Wireless — electromagnetic waves.

MODULE 4
Physical Layer
Objective: Explain how physical layer protocols, services, and network media support communications across data networks.
4.1 — Purpose & 4.2 — Characteristics
THE PHYSICAL LAYER

Transports bits across network media. Accepts a complete frame from the Data Link Layer, encodes it as signals, and transmits it — the last step in encapsulation.

PHYSICAL COMPONENTS

Hardware devices, media, and connectors that transmit signals — NICs, interfaces, connectors, cable materials and designs. All specified in standards.

ENCODING

Converts stream of bits into a format recognizable by the next device. Examples: Manchester4B/5B8B/10B

SIGNALING

How "1" and "0" are represented on the physical medium. Varies by medium — electrical (copper), light pulses (fiber), microwave (wireless).

GOODPUT

Measure of usable data transferred. Goodput = Throughput − traffic overhead

4.3 — Copper Cabling
Most common cabling — inexpensive, easy to install, low resistance. Limitations: Attenuation (signal weakens over distance), EMI/RFI interference, Crosstalk.
UTP — Unshielded Twisted Pair

Most common networking media. RJ-45 connectors. Relies on cancellation (opposite polarity wires) and variation in twists per foot to limit crosstalk.

STP — Shielded Twisted Pair

Better noise protection than UTP. More expensive and harder to install. Also uses RJ-45. Has braided/foil shield per pair for EMI/RFI protection.

COAXIAL CABLE

Layers: outer jacket → woven copper braid (shield) → flexible plastic insulation → copper conductor. Used for wireless antenna connections and cable internet installations.

Number Systems
Objective: Calculate numbers between decimal, binary, and hexadecimal systems.
5.1 — Binary Number System
Binary & IPv4: Binary = 1s and 0s (bits). Each IPv4 address = 32 bits ÷ 4 octets (8 bits each), separated by dots. Converted to dotted decimal for human use. Routers and computers only understand binary.
Binary Positional Values
Position (2ⁿ)76543210
Value1286432168421
Binary → Decimal
Convert 11000000.10101000.00001011.00001010
11000000 = 128+64 = 192  |  10101000 = 128+32+8 = 168  |  00001011 = 8+2+1 = 11  |  00001010 = 8+2 = 10
Result: 192.168.11.10
Decimal → Binary Algorithm
  • Start at the 128 position (most significant bit).
  • If decimal value ≥ current position → write 1, subtract, move right.
  • If decimal value < current position → write 0, move right.
  • Repeat through position 1. Remaining positions get 0.
Example: Convert 168 to binary
168≥128 → 1, rem 40  |  40<64 → 0  |  40≥32 → 1, rem 8  |  8<16 → 0  |  8=8 → 1, rem 0  |  0,0,0 for positions 4,2,1
Result: 10101000
5.2 — Hexadecimal Number System
Hexadecimal — base 16 system using 0–9 and A–F. Used for IPv6 addresses and MAC addresses. 1 hex digit = 4 bits. Each group of 4 hex digits in an IPv6 address is called a hextet. IPv6 = 128 bits = 32 hex digits.

Decimal → Hex (3 Steps)
  • Step 1: Convert decimal to 8-bit binary string.
  • Step 2: Split binary into groups of 4 (from the right).
  • Step 3: Convert each 4-bit group to its hex digit.
168 → Hex: 168 = 10101000 → 1010 | 1000 → A | 8 → A8
Hex → Decimal (3 Steps)
  • Step 1: Convert each hex digit to 4-bit binary string.
  • Step 2: Create 8-bit binary grouping from the right.
  • Step 3: Convert each 8-bit grouping to decimal.

MODULE 6
Data Link Layer
Objective: Explain how media access control in the data link layer supports communication across networks.
6.1 — Purpose of the Data Link Layer
RESPONSIBILITIES
  • Communications between end-device NICs
  • Allows upper layers to access physical media
  • Encapsulates L3 packets (IPv4 & IPv6) into L2 Frames
  • Error detection — rejects corrupt frames
LLC SUBLAYER (IEEE 802.2)

Communicates between networking software (upper layers) and device hardware (lower layers). Identifies which L3 protocol is used for the frame.

MAC SUBLAYER (802.3/802.11/802.15)

Responsible for data encapsulation and media access control. Provides data link layer addressing.

AT EACH HOP (ROUTER)
  1. Accept frame from the medium
  2. De-encapsulate (expose the packet)
  3. Re-encapsulate into a new frame
  4. Forward on the next network segment
Standards defined by: IEEE · ITU · ISO · ANSI
6.2 — Topologies & Access Control
WAN: POINT-TO-POINT

Simplest and most common WAN topology. Permanent link between two endpoints. Protocols can be very simple as frames only travel between two nodes.

WAN: HUB AND SPOKE

Central site interconnects branch sites via point-to-point links. Similar to a star topology.

WAN: MESH

High availability — every end system connected to every other. Most redundant but most complex.

LAN: STAR / EXTENDED STAR

Most common modern LAN topology. Easy to install, scalable, and easy to troubleshoot.

CSMA/CD (Collision Detection)

Legacy Ethernet LANs. Half-duplex. Devices that collide → detect → wait a random period → retransmit.

CSMA/CA (Collision Avoidance)

IEEE 802.11 WLANs. Half-duplex. Device includes duration needed for transmission so others know to wait — preventing collisions proactively.

Half-DuplexFull-Duplex
Only one device can send OR receive at a time. Used on WLANs and legacy Ethernet hubs. Requires CSMA/CD or CSMA/CA.
Both devices simultaneously transmit and receive. Ethernet switches operate in full-duplex. NO need for CSMA/CD.

7.2 — Ethernet MAC Address
MAC Address: 48-bit binary value expressed as 12 hexadecimal digits (6 bytes). Physically embedded into the Ethernet NIC. Must be globally unique. Vendors register with IEEE for a unique 24-bit OUI (Organizationally Unique Identifier).
UNICAST MAC

Unique address for one-to-one delivery. Source MAC is always unicast.

ARP — resolves IPv4 → MAC.
ND (Neighbor Discovery) — resolves IPv6 → MAC.

BROADCAST MAC

Received by every device on the LAN. Flooded out all switch ports except incoming. NOT forwarded by a router.

FF-FF-FF-FF-FF-FF
MULTICAST MAC

Received by a specific group of devices. Flooded out all ports except incoming.

IPv4: 01-00-5E-xx-xx-xx
IPv6: 33-33-xx-xx-xx-xx
Source is always unicast.

Frame Processing: NIC receives frame → checks destination MAC against its own. No match → discard. Match (or broadcast/multicast group member) → pass up the OSI layers for de-encapsulation.
7.3 — The MAC Address Table (CAM Table)
MAC Address Table: A Layer 2 switch makes forwarding decisions solely on Layer 2 MAC addresses — completely unaware of Layer 3 data. Table is empty when switch first powers on.
LEARNING (Source MAC)

Every incoming frame: examine source MAC + incoming port.

  • MAC not in table → add it with port number
  • MAC exists → refresh the 5-minute timer
  • MAC exists on a different port → update to new port
FORWARDING (Destination MAC)
  • MAC in table → forward out that specific port
  • MAC NOT in table → flood all ports except incoming (unknown unicast)
  • Broadcast or multicast → also flood all ports except incoming

Once MAC is learned → switch can filter and forward only to the correct port.

MODULE 8
Network Layer
Objective: Explain how the network layer uses IP protocols for reliable communications.
8.1 — Network Layer Characteristics
THE NETWORK LAYER

Provides services to allow end devices to exchange data. IPv4 and IPv6 are the principal network layer protocols. Performs four basic operations: Addressing end devices → Encapsulation → Routing → De-encapsulation.

IP ENCAPSULATION

IP encapsulates the transport layer segment. Can use either IPv4 or IPv6 without impacting the Layer 4 segment. IP addressing does NOT change from source to destination (NAT is an exception, covered later).

IP Characteristics — 3 Key Properties: IP is designed with low overhead and is described as Connectionless, Best Effort, and Media Independent.
PropertyDescription
ConnectionlessNo connection is established with the destination before sending. No synchronization or acknowledgments needed. Destination receives the packet when it arrives — no pre-notification sent.
Best EffortNo guarantee of delivery. No mechanism to resend lost data. IP does not expect acknowledgments or know if the other device received the packet. Reduced overhead as a result.
Media IndependentDoes not concern itself with the type of frame (Data Link) or media (Physical). IP can be sent over copper, fiber, or wireless. The network layer establishes the MTU (Maximum Transmission Unit).
Fragmentation (IPv4 only): When a router goes from Ethernet to a slower WAN with a smaller MTU, Layer 3 may split the IPv4 packet into smaller units. Fragmenting causes latency. IPv6 does NOT fragment packets.
8.2 — IPv4 Packet Header Fields
IPv4 Header is in binary, contains several fields of information, and is read left to right — 4 bytes per line. The two most important fields are the source and destination IP addresses. Total header size: 20 bytes minimum.
FieldDescription
VersionIPv4 = 0100 (4-bit field). Identifies IP version in use.
Differentiated Services (DS)Used for QoS — DiffServ (DSCP) field or older IntServ (ToS/Type of Service).
Header ChecksumDetects corruption in the IPv4 header.
Time-to-Live (TTL)Layer 3 hop count. Decremented by 1 at each router. When it reaches zero, the router discards the packet.
ProtocolIdentifies the next level protocol: ICMP (1), TCP (6), UDP (17), etc.
Source IPv4 Address32-bit source address.
Destination IPv4 Address32-bit destination address.
Identification / Flag / Fragment OffsetUsed for fragmentation and reassembly of packets.
8.3 — IPv6 Packet
IPv4 Limitations that led to IPv6: IPv4 address depletion · Lack of end-to-end connectivity (NAT workaround) · Increased network complexity from NAT (latency + troubleshooting issues).
IPv6 IMPROVEMENTS
  • Increased address space — 128-bit (vs 32-bit IPv4). ~340 undecillion addresses.
  • Improved packet handling — simplified header with fewer fields.
  • Eliminates need for NAT — massive address space removes need for private addressing.
IPv6 HEADER FACTS

Simplified but fixed at 40 bytes. Fields removed from IPv4 to improve performance: FlagFragment OffsetHeader Checksum

8.4 — How a Host Routes
HOST FORWARDING DECISION

Packets are always created at the source. Each host creates its own routing table. A host can send to:

  • Itself — 127.0.0.1 (IPv4) or ::1 (IPv6)
  • Local Hosts — destination on same LAN
  • Remote Hosts — destination on a different LAN
LOCAL vs REMOTE DETERMINATION

IPv4: Source uses its own IP + subnet mask + destination IP to determine if local or remote.

IPv6: Source uses the network address and prefix advertised by the local router.

Remote traffic → forwarded to the default gateway.

DEFAULT GATEWAY (DGW)

A router or Layer 3 switch. Must have an IP in the same range as the LAN. Can accept data from LAN and forward traffic off the LAN. Without a DGW, traffic cannot leave the local network.

DGW DISCOVERY

IPv4: Learned statically or via DHCP.
IPv6: Learned via Router Solicitation (RS) or configured manually.
The DGW is a static route of last resort in the routing table.

View host routing table on Windows: route print or netstat -r — displays Interface List, IPv4 Routing Table, and IPv6 Routing Table.
8.5 — Introduction to Routing
Router Packet Forwarding: (1) Packet arrives → de-encapsulate L2 header/trailer. (2) Examine destination IPv4 address → search routing table for best match. (3) Re-encapsulate into new frame → forward to next hop.
Route TypeDescription
Directly Connected (C / L)Automatically added when interface is active and has addressing. C = network, L = local interface IP.
Remote — Static (S)Manually configured by administrator. Must be manually updated on topology change. Good for small non-redundant networks.
Remote — Dynamic (O, D…)Learned via routing protocols (OSPF, EIGRP, etc.). Automatically discovers, maintains, and updates routes. Finds new best paths after topology changes.
Default Route (S*)Forwards all traffic to a specific direction when no match exists in routing table — route of last resort.

MODULE 9
Address Resolution
Objective: Explain how ARP and ND enable communication on a network.
9.1 — MAC and IP
Two primary addresses on an Ethernet LAN device: Layer 2 MAC address (physical — NIC to NIC on same network) and Layer 3 IP address (logical — source to destination across networks).
Destination on Same NetworkDestination on Remote Network
Destination MAC address = MAC of the destination device itself. ARP resolves it directly.
Destination MAC address = MAC of the default gateway (router interface). ARP resolves the gateway's MAC.
Source IP ↔ Destination IP: same network portion. Layer 2 goes directly to the target device.
Source IP ↔ Destination IP: different networks. Frame goes to router; router handles forwarding to final destination.
IPv4 ADDRESS RESOLUTION

ARP — Address Resolution Protocol. Associates an IPv4 address with the device's MAC address. Used to find the MAC for local communication or the default gateway.

IPv6 ADDRESS RESOLUTION

ICMPv6 / ND — Neighbor Discovery Protocol. Associates an IPv6 address with the device's MAC address. Replaces ARP for IPv6 networks.

9.2 — ARP (Address Resolution Protocol)
ARP provides two basic functions: (1) Resolving IPv4 addresses to MAC addresses. (2) Maintaining an ARP table (cache) of IPv4-to-MAC address mappings.
ARP FunctionBehavior
Search ARP tableDevice checks its ARP table for the destination IPv4 or default gateway IPv4 address.
MAC foundCorresponding MAC used as destination MAC in the frame — no ARP request needed.
MAC not foundDevice sends an ARP Request — broadcast to FF-FF-FF-FF-FF-FF on the local network.
ARP ReplyThe device with the matching IP responds with its MAC address — unicast reply.
Table updatedRequester stores the IP-to-MAC mapping in its ARP cache for future use.
ARP TABLE AGING

Entries are not permanent. Removed when the ARP cache timer expires (duration varies by OS). Can also be manually removed by an administrator.

VIEW ARP TABLE

On a Cisco router: show ip arp
On a Windows PC: arp -a

Shows IP-to-MAC mappings and their type (dynamic/static).

ARP Issues: ARP requests are broadcasts — received/processed by every device on the LAN. Excessive ARP broadcasts can reduce performance. ARP replies can be spoofed by a threat actor (ARP poisoning / ARP spoofing attack). Enterprise switches include mitigation techniques against ARP attacks.
9.3 — IPv6 Neighbor Discovery (ND)
IPv6 Neighbor Discovery (ND) Protocol provides: Address resolution · Router discovery · Redirection services. Uses ICMPv6 messages instead of ARP.
Message TypeAbbreviationPurpose
Neighbor SolicitationNSDevice-to-device. Used to resolve IPv6 address → MAC address. Sent to a multicast address.
Neighbor AdvertisementNADevice-to-device. Reply to NS — provides the MAC address of the responding device.
Router SolicitationRSDevice-to-router. Used to discover routers on the link.
Router AdvertisementRARouter-to-device. Routers advertise themselves and network prefix info.
Redirect—Router-to-device. Used for better next-hop selection.
ND Address Resolution Process: PC1 sends an ICMPv6 Neighbor Solicitation to a special multicast address asking who has a specific IPv6 address. The target device (PC2) responds with a Neighbor Advertisement containing its MAC address. ICMPv6 NS messages use special Ethernet and IPv6 multicast addresses.

10.3 — Configure the Default Gateway
DEFAULT GATEWAY ON A HOST

Used when a host sends a packet to a device on another network. The DGW address is typically the router interface attached to the local LAN. The IP of the host and router interface must be in the same network.

DEFAULT GATEWAY ON A SWITCH

Needed for remote management of the switch from another network. Configure with the ip default-gateway command.

! Configure default gateway on a switch for remote management S1(config)# ip default-gateway 192.168.10.1
Key Rule: Without a default gateway (or with a bad one), a device's traffic cannot leave the local LAN. All devices on a LAN need the DGW of the router to send traffic remotely.

MODULE 11
IPv4 Addressing
Objective: Calculate an IPv4 subnetting scheme to efficiently segment your network.
11.1 — IPv4 Address Structure
An IPv4 address is a 32-bit hierarchical address composed of a network portion and a host portion.
SUBNET MASK

Identifies the network/host portions by comparing bits from left to right. Uses logical ANDing: only 1 AND 1 results in 1.

PREFIX LENGTH

The number of bits set to 1 in the mask, written in slash notation (e.g., /24).

Address TypeHost Bit LogicDescription
Network AddressAll 0sIdentifies the network itself.
First HostAll 0s and a 1The first assignable IP in the subnet.
Last HostAll 1s and a 0The last assignable IP in the subnet.
BroadcastAll 1sUsed to communicate with all hosts on the network.
11.2 — Unicast, Broadcast, and Multicast
UNICAST

One-to-one communication; sending a packet to a single destination.

BROADCAST

One-to-all; destination address is 255.255.255.255.

MULTICAST

One-to-group; sent to a specific multicast group address (e.g., 224.0.0.0 range).

11.3 — Types of IPv4 Addresses
Public addresses are globally routed, while Private addresses (RFC 1918) are used internally and translated via NAT for internet access.
Private RangeSpecial Use Addresses
10.0.0.0/8Loopback: 127.0.0.1 (Tests operational TCP/IP).
172.16.0.0/12Link-Local (APIPA): 169.254.0.0/16.
192.168.0.0/16Self-assigned when no DHCP server is found.
Legacy Classful Addressing: Originally divided into Classes A, B, and C. This was replaced by classless addressing to reduce address waste.
11.4 — Network Segmentation & Subnetting
WHY SUBNET?
  • Reduces broadcast traffic.
  • Improves network performance.
  • Allows implementation of security policies.
VLSM (Variable Length)

The process of "subnetting a subnet" to avoid wasting host addresses. Always satisfy the requirements of the largest subnet first.

Segmentation Criteria: Organization can segment by Location, Group/Function, or Device Type.

MODULE 12
IPv6 Addressing
Objective: Implement an IPv6 Addressing scheme.
12.1 — IPv4 Issues & Coexistence
IPv4 addresses are exhausted. IPv6 provides a 128-bit address space and fixes IPv4 limitations.
DUAL STACK

Devices run both IPv4 and IPv6 protocol stacks simultaneously.

TUNNELING

Transporting an IPv6 packet over an IPv4 network by encapsulating it inside an IPv4 packet.

TRANSLATION (NAT64)

Allows IPv6-enabled devices to communicate with IPv4-enabled devices.

12.2 — Address Representation
Format: x:x:x:x:x:x:x:x (8 hextets of 16 bits each).
RULE 1: OMIT LEADING ZEROS

Example: 0db8 becomes db8. 00ab becomes ab.

RULE 2: DOUBLE COLON (::)

Replaces contiguous string of all-zero hextets. Used only once per address.

12.3 — IPv6 Address Types
TypeDescription
UnicastUniquely identifies an interface on an IPv6 device.
MulticastSends a single packet to multiple destinations.
AnycastAssigned to multiple devices; routed to the nearest one.
IPv6 does NOT use broadcast addresses.
GLOBAL UNICAST (GUA)

Globally unique, internet-routable (similar to public IPv4). Usually begins with 2xxx: or 3xxx:.

LINK-LOCAL (LLA)

Required for every device; used for local link communication only. Range: fe80::/10.

GUA Structure: Global Routing Prefix (48 bits) + Subnet ID (16 bits) + Interface ID (64 bits).
12.4 – 12.6 — Addressing Configuration
! Static GUA and LLA Configuration Router(config-if)# ipv6 address 2001:db8:acad:1::1/64 Router(config-if)# ipv6 address fe80::1 link-local
Dynamic Addressing Methods
  • SLAAC: Device creates its own GUA using Router Advertisement (RA) prefix.
  • Stateless DHCPv6: SLAAC for GUA, DHCPv6 for DNS/domain info.
  • Stateful DHCPv6: DHCPv6 server provides all addressing info (no SLAAC).
Interface ID Generation: Can be EUI-64 (uses MAC address + fffe) or Randomly Generated
MODULE 13
ICMP
Objective: Use various tools to test network connectivity.
13.1 — ICMP Messages
ICMP provides feedback about issues related to IP packet processing.
MessagePurpose
Echo Request/ReplyTests host reachability (used by Ping).
Dest. UnreachableNotifies source that a service or destination is unavailable.
Time ExceededSent when TTL (IPv4) or Hop Limit (IPv6) reaches 0.
ICMPv6 Specific Messages (Neighbor Discovery)
ROUTER RS/RA

RS: Host seeks a router.
RA: Router provides addressing/network info every 200s.

NEIGHBOR NS/NA

NS: Sent to find a device's MAC address or for DAD.
NA: Response containing the MAC address.

13.2 — Ping and Traceroute Tests
PING

Uses Echo Request/Reply to test connectivity and measure round-trip time.

  • Loopback: 127.0.0.1 / ::1 (tests stack).
  • Default Gateway: Tests local LAN access.
TRACEROUTE (tracert)

Lists all hops to a destination. Logic: Starts with TTL=1, increments for each hop until destination is reached.

Note: An asterisk (*) in traceroute indicates a lost packet or a router configured not to reply.
MODULE 14
Transport Layer
Objective: Compare the operations of transport layer protocols in supporting end-to-end communication.
14.1 — Transportation of Data
The transport layer is responsible for logical communications between applications running on different hosts. It acts as the link between the application layer and the lower layers responsible for network transmission.
RESPONSIBILITIES
  • Tracking individual conversations
  • Segmenting data and reassembling segments
  • Identifying applications using port numbers
  • Multiplexing multiple conversations over the network
PROTOCOL SELECTION

TCP: Used when reliability is required (e.g., HTTP, Email, FTP).
UDP: Used when speed/low overhead is prioritized (e.g., VoIP, DNS, Streaming).

14.2 — TCP Overview (Connection-Oriented)
TCP is a stateful protocol that ensures reliable delivery, same-order delivery, and supports flow control.
TCP Header FieldDescription
Sequence NumberUsed for data reassembly/ordering.
Acknowledgment NumIndicates data received and next byte expected.
Window SizeIndicates number of bytes that can be accepted at once.
Control BitsFlags (SYN, ACK, FIN, RST, etc.) that manage the session.
3-WAY HANDSHAKE

1. SYN (Initiate)
2. SYN, ACK (Synchronize & Acknowledge)
3. ACK (Finalize)

SESSION TERMINATION

Uses a four-way exchange: FIN (sender) -> ACK (receiver) -> FIN (receiver) -> ACK (sender).

14.3 — UDP Overview (Connectionless)
UDP CHARACTERISTICS

Known as a best-effort delivery protocol because it does not require acknowledgments or resend lost data. It has very low overhead.

UDP HEADER

Simple 8-byte header with only four fields: Source Port, Destination Port, Length, and Checksum.

14.4 — Port Numbers & Sockets
Socket: The combination of an IP address and a port number. It enables multiple processes on a single client to be distinguished from each other.
Port GroupRange Usage
Well-Known0 - 1,023Common services (HTTP, FTP, SMTP).
Registered1,024 - 49,151Assigned by IANA to specific applications.
Private/Dynamic49,152 - 65,535Ephemeral ports assigned by the client OS.
MODULE 15
Application Layer
Objective: Explain the operation of application layer protocols in providing support to end-user applications.
15.1 — Presentation and Session Layers
The OSI Application, Presentation, and Session layers are merged into the single TCP/IP Application layer.
PRESENTATION LAYER

Handles data formatting, compression, and encryption/decryption (e.g., JPEG, GIF, MKV).

SESSION LAYER

Creates and maintains dialogs between source and destination applications.

15.2 — Network Models
Client-Server ModelPeer-to-Peer (P2P)
Dedicated server responds to client requests.
Devices act as both clients and servers simultaneously.
15.3 — Web and Email Protocols
ProtocolTypeUsage
HTTPTCP 80GET (Request), POST (Upload data), PUT (Upload resource).
SMTPTCP 25Used by clients to send email to a server.
POP3TCP 110Retrieves email; messages are typically deleted from the server.
IMAPTCP 143Retrieves email; copies are stored on the server until manual deletion.
15.4 — IP Addressing Services
DNS (UDP/TCP 53)

Translates domain names (FQDNs) into IP addresses. Uses a hierarchy of Root, Top-Level (TLD), and Second-Level domains.

DHCP (UDP 67/68)

Automates IPv4 address assignment. Process: Discover -> Offer -> Request -> Acknowledgment (DORA).

15.5 — File Sharing
FTP: Uses TCP port 21 for control and port 20 for actual data transfer.
SMB: A client-server request-response protocol used for long-term file and printer sharing.
MODULE 16
Network Security Fundamentals
Objective: Configure switches and routers with device hardening features.
16.1 — Threats and Vulnerabilities
THREAT TYPES

Information Theft: Stealing data.
Identity Theft: Stealing credentials.
Data Manipulation: Altering info.
Service Disruption: Preventing authorized use.

VULNERABILITIES

Technological: Protocol or OS weaknesses.
Configuration: Unsecured accounts/default settings.
Security Policy: Lack of written rules/politics.

Physical Security: Protect against Hardware damage, Environmental extremes, Electrical spikes/loss, and Maintenance errors (poor cabling/labeling).
16.2 — Network Attacks
MalwareDescription
VirusesAttached to a program; requires human action to spread.
WormsStandalone software; self-replicates without human help.
Trojan HorsesLegitimate-looking software that contains harmful code.
RECONNAISSANCE

Discovery/mapping of systems and vulnerabilities using tools like nslookup or whois.

ACCESS ATTACKS

Exploiting vulnerabilities in services (FTP, web) to gain entry. Includes Password attacks, Trust exploitation, and Man-in-the-middle.

DoS / DDoS

Disabling systems by consuming resources. DDoS uses a botnet of "zombie" hosts to coordinate attacks.

16.3 — Mitigation Techniques
Defense-in-Depth: A layered approach using tandem devices/services like Firewalls, IPS, and VPNs.
AAA ServiceFunction
AuthenticationWho are you? (Identification)
AuthorizationWhat can you do? (Permissions)
AccountingWhat did you do? (Logging)
Firewall Types
  • Packet Filtering: Based on IP or MAC addresses.
  • Application Filtering: Based on port numbers.
  • Stateful Packet Inspection (SPI): Blocks unsolicited packets; requires legitimate responses to internal requests.
16.4 — Device Security & Hardening
! Cisco Password Hardening Router(config)# security password min-length 8 Router(config)# service password-encryption Router(config)# login block-for 120 attempts 3 within 60 Router(config-line)# exec-timeout 5 30
Secure Shell (SSH) Requirements
  1. Unique Hostname
  2. IP Domain Name (ip domain-name name)
  3. RSA Keys (crypto key generate rsa - min 1024 bits recommended)
  4. Local Username/Password Database
  5. Line VTY Authentication (login local)
  6. VTY Transport (transport input ssh)

The Troubleshooting Process
  • 1. Identify the Problem (Conversation with user).
  • 2. Establish a Theory of Probable Causes.
  • 3. Test Theory to determine cause.
  • 4. Establish a Plan of Action and Implement Solution.
  • 5. Verify full functionality and implement preventive measures.
  • 6. Document findings, actions, and outcomes.
